# Notation Consistency

- each symbol has one meaning
- each concept has one preferred name
- avoid redefining symbols across sections
- keep theorem notation aligned with algorithm notation
- keep problem-definition answer objects aligned with pseudocode return values
- keep base/current/logical state superscripts and time subscripts explicit
- distinguish candidate sets, candidate groups, verified sets, and final answers
- distinguish access, scan, verification, build, rebuild, and flush costs
- verify units and dimensions in complexity terms

Search the full source for deprecated names after terminology changes. Include section labels, algorithm names, figure descriptions, appendix headings, and complexity variables.
