# DB Figure Design

Use this skill for claim-driven figure planning and figure QA.

## Workflow

1. Load `references/figure_story_rules.md`, `references/figure_types.md`, and `references/visual_style_rules.md`.
2. Load `.references/distilled/iterative_revision_lessons.md` when revising a multi-figure paper.
3. For each target claim, define one primary figure/table.
4. Select figure type based on reader question.
5. Choose single- or double-column width from information density, not page filling.
6. Ensure captions state takeaway and conditions concisely.
7. Validate readability and placement under two-column constraints.

## Output Contract

1. `Figure Plan`
2. `Claim Linkage`
3. `Caption Drafts`
4. `Visual QA Issues`

## Rules

- Avoid decorative figures without claim linkage.
- Include axis units and comparison conditions explicitly.
- Prefer structural diagrams over flowcharts for index state and storage layout.
