# DB Figure Design

Use this skill for claim-driven figure planning and figure QA.

## Workflow

1. Load `references/figure_story_rules.md`, `references/figure_types.md`, and `references/visual_style_rules.md`.
2. For each target claim, define one primary figure/table.
3. Select figure type based on reader question.
4. Ensure captions state takeaway and conditions.
5. Validate readability under two-column constraints.

## Output Contract

1. `Figure Plan`
2. `Claim Linkage`
3. `Caption Drafts`
4. `Visual QA Issues`

## Rules

- Avoid decorative figures without claim linkage.
- Include axis units and comparison conditions explicitly.
