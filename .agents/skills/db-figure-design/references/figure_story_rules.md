# Figure Story Rules

- one figure answers one core question
- architecture figures show state, ownership, and data relationships; flowcharts show temporal sequence
- teaser figure reveals the core method and is referenced near the first proposal paragraph
- caption includes setup + key takeaway without extended baseline argument
- legend and labels are unambiguous
- figure is interpretable without surrounding paragraph
- figure placement is close to the text that asks and answers its question
- accessibility description names panels, encodings, and visible trends

For experiments, prefer grouped bars for cross-dataset summaries and lines for scale or parameter sweeps. Split panels when one axis or orientation makes a baseline unreadable.
